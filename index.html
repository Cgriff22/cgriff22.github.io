How This Site Reaches You

🌐

DNS Resolution

The Domain Name System translates our human-readable domain into a machine-routable IP address. Without DNS, you'd need to memorize 185.199.108.153 instead of a clean URL.

📡

IP Addressing

Every device on the internet is identified by an IP address — IPv4 (32-bit) or IPv6 (128-bit). GitHub Pages resolves to multiple IPv4 addresses for load distribution and redundancy.

🔒

HTTPS / TLS

HTTPS wraps HTTP inside a TLS tunnel, encrypting all data in transit. GitHub Pages enforces HTTPS by default, providing certificate-based authentication and data integrity.

📦

TCP/IP Stack

Data moves through layers: Application (HTTP), Transport (TCP), Internet (IP), and Network Access (Ethernet/Wi-Fi). Each layer encapsulates the one above it in headers and trailers.

DNS Resolution in Action

How a Browser Finds This Site

When you type our URL into your browser, a chain of lookups occurs before a single pixel renders on screen. DNS is the internet's phonebook — converting domain names to IP addresses.

  1. Browser checks its local DNS cache for a cached record
  2. If no cache hit, the OS resolver queries the configured DNS server
  3. The recursive resolver walks the hierarchy: root → .io TLD → GitHub's authoritative nameserver
  4. An A record (IPv4) or AAAA record (IPv6) is returned
  5. The browser opens a TCP connection to the resolved IP

HTTP vs HTTPS

HTTP transfers data in plaintext — anyone on the network path can read or modify it. HTTPS adds a TLS handshake that encrypts the connection, verifies the server's identity via certificates, and ensures data integrity.

Feature HTTP HTTPS
Port 80 443
Encryption None TLS 1.2 / 1.3
Certificate Not required Required (X.509)
Data Integrity Vulnerable to MITM Cryptographic verification
SEO Impact Penalized by search engines Ranking boost
Status Code Example 200 OK, 404 Not Found 200 OK, 301 Redirect to HTTPS

Securing the Deployment

GitHub Pages enforces HTTPS by default. Beyond that, we implement additional security headers to harden the site against common web attacks.

ENABLED

HTTPS Enforcement

All HTTP requests are 301-redirected to HTTPS. HSTS header prevents protocol downgrade attacks.

ENABLED

HSTS Header

Strict-Transport-Security with max-age=31536000 instructs browsers to always use HTTPS for future visits.

ENABLED

X-Content-Type-Options

Set to "nosniff" to prevent browsers from MIME-sniffing responses away from the declared content type.

ENABLED

X-Frame-Options

Set to "DENY" to prevent clickjacking by blocking the site from being loaded in iframes.
